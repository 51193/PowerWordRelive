输出格式：每行一条指令，格式如下：

```
task|append|简述|详细描述                 新增任务。简述是任务标识，需在进行中的任务中唯一。
task|remove|简述                          删除任务（仅限需要直接移除的错误/无效条目，正常结束用 finish）。
task|edit|简述|详细描述                   修改任务的详细描述，简述不变。
task|replace|旧简述|新简述|新描述         同时修改简述和详细描述。
task|finish|简述|状态                    标记任务状态变更。状态取值：complete（完成）、fail（失败）、discard（放弃）。
EMPTY                                    本回合无任何变化。
```

约束：
- 简述限制在50字以内
- 详细描述无长度限制，应包含任务委托人、目标、地点、奖惩等完整信息
- 不要输出任何解释性文字、Markdown、JSON 或格式标记
- 管道符 `|` 可能出现在简述或描述中，解析器会正确处理
- finish 的状态字段仅接受 complete、fail、discard
- 每行只能有一条指令；无法解析的行会被静默忽略
- 如果输出 EMPTY，之后的行都将被忽略
