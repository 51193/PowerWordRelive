处理步骤：

1. 阅读精炼结果和故事进展，了解最新的情节发展和团队动向。
2. 检查「进行中的任务」列表，判断：
   - 是否有任务已经完成、失败或应该放弃？如果有，使用 finish 指令标记状态。
   - 是否有任务描述需要补充或修正？如果有，使用 edit 或 replace 指令更新。
   - 是否有任务已经过时或不再相关？如果有，使用 remove 指令删除。
3. 从精炼结果中提取新出现的任务目标（NPC 委托、新线索、团队主动提出的计划等），使用 append 指令追加。
4. 仅对有变化的内容输出操作。若完全无需修改，输出 `EMPTY`。

注意事项：
- 精炼结果和故事进展是近期对话的**局部窗口**，仅供参考。它们不包含完整历史，不要因为窗口内没看到某个人而认为任务已经不存在。
- 新任务始终使用 `append` 追加。
- 修改任务用 `edit`（只改详细描述）或 `replace`（同时修改简述和描述）。
- 完成任务用 `finish` + 状态标记，不要用 remove 删除已完成/失败/放弃的任务。
- finish 状态的取值：`complete`（成功完成）、`fail`（明确失败）、`discard`（长时间未提及/已放弃）。
- 请积极对明显已过时或团队不再关心的任务进行清理，使用 finish + discard。
- 简述是任务的唯一标识，修改简述时使用 replace 指令。
