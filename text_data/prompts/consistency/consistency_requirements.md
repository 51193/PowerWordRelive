处理步骤：

1. 阅读精炼结果、故事进展和进行中的任务，了解最新剧情动向。
2. 检查一致性表格，判断：
   - 是否有新出现的人物、地点、物品、组织需要收录？如果有，使用 append 指令新增。
   - 是否有条目的描述需要补充或修正（如人物状态变化、地点被摧毁、物品归属转移等）？如果有，使用 edit 指令更新。
   - 是否有条目已彻底过时（角色死亡无法复活、地点被摧毁无法返回、物品永不可恢复等）？如果有，使用 remove 指令删除。
3. 优先保证信息完整：宁可多记录一个潜在有用的条目，也不要漏掉可能被未来 Agent 查询的内容。
4. 但也要积极清理不再需要的条目，保持表格精简。
5. 仅对有变化的内容输出操作。若完全无需修改，输出 `EMPTY`。

注意事项：
- 条目的名称（name）是唯一标识。新增时请勿与已有条目重名。
- 精炼结果和故事进展是近期对话的局部窗口。一致性表格可能包含更早记录的条目，不要因为窗口内没看到就删除。
- 一致性表格的存储空间有限，请主动清理不再需要的条目。
- 使用编辑（edit）来修正信息不准确或过时的条目。
